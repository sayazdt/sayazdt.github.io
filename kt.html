Kernel trick

By Farid Tagiyev Machine, Learning, Classifier Contact

The World is a place where ideal conditions are rare. In some cases, non-ideal cases bring a couple of problems which data scientists and machine learning experts have to solve. Here we will talk about one of such problems, why this problem is important for business, and how it can be solved. 

 

            When it comes to classification, there are usually 2 types of data: linearly and non-linearly separable. If data is linearly separable, we consider it as unique situations and it seems piece of cake to separate such data with the help algorithms such as logistic regression, perceptron, support vector machine, and others. (for simplicity purposes consider binary classification problems). However, if data is non-linearly separable, then we have a problem, Before moving to the solution of this issues, let’s see why we should actually care about it.

 

            Ability to classify data is extremely crucial for industries. Classification algorithms widely used for email spam filtering, fraud detection, weather forecasting, image detection, health care, NLP, sentiment analyses, and many others. So, if data scientist or machine learning expert faces issues with non-linearly separable data set, he/she has to solve it. But what are solutions?

 


 

           As we’ve already mentioned, one of the algorithms to solve classification problem is Support Vector Machine (SVM). So, SVM by itself cannot separate non-linearly separable data, but there is an idea usually used along with SVM which is called kernel trick. Let me visually explain what this trick does. Image we have such data:

As we can see, blue data points mean normal patient and red ones those who have cancer. It is typical example of binary non-linearly separable classification problem. So, how kernel trick works here? Let’s consider next picture of the same data set. 



           So, what have changed? Now we can see that data points are linearly separable by plane. So, what kernel trick essentially does is that it maps original data points with the help of special functions into higher dimension until eventually it becomes linearly separable. Nevertheless, it is not the only advantage of kernel trick. Another issue here raises which is related with computation cost. When we talk about higher dimensional data points (which is usually the case), it is pretty difficult even for machine to perform complicated computations with lots of variables. So, another beauty of kernel trick is that it reduces all of these complexities without moving into higher dimension. So, simply speaking kernel trick performs computation in original feature space, without making computations for higher dimension. 

 

            In conclusion, considering all said above, we can surely say that kernel trick is another powerful tool used by data scientists to make better classifications in case of non-linearly separable data points. As it was already mentioned, kernel trick usually used along with Support Vector Machine, so let me know if you want to see new post regarding discussion about this algorithm.
